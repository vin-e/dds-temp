# Events

Stencil events emit CustomEvents as it's eventing type. Here are a few tips and tricks when using events to communicate between components and the DOM.

# Event Naming

A standard way to interact with events on an element is to prefix it with on. A click event would then be onclick. Well this does not work for naming a event emitted in a stencil component. If we named our event to be onClickButton when we reference it later it would then be onOnClickButton.

Here are a few rules we should adhere to when naming events:

  • Do not prefix the event with on
  • In general, the event has typically already happened. We should write the event in past tense. i.e. buttonClicked

# Standard Browser Events (i.e. focus and blur)

The default browser events for focus and blur do not bubble up through the DOM elements. Your only chance to catch those events are directly on the element. If the component you are build needs to propagate the events you need to capture and re-emit.

This is where the real fun begins with focus and blur. In stencil events you cannot emit browser event names. If you attempt to do so then a build error will be triggered. The solution for emitting events, like focus and blur, is to use a standard event.

# Example

First we need to ensure we have a reference to the host element that the event will be emitting from

@Element() hostElement: HTMLElement

Then we will create a private function that handles our event. Here we will emit and standard DOM event

private onBlurHandler() {
  this.hostElement.dispatchEvent(new Event('blur'))
}

Finally we attach to the element where we are listening to the blur event and then call the blur handler method.

<input
  value="value here"
  onBlur={this.onBlurHandler.bind(this)} />

# Mixing standard browser events and Stencil events

Stencil's Event is named exactly like the DOM event. If you need to use both types of events in your component then we should alias the stencil event during import.

import { h, Props, Event as StencilEvent, EventEmitter } from '@stencil/core'

Then using the event later would look like:

@StencilEvent() buttonClicked: EventEmitter
