# Development Environment

# Node

It appears that older versions of Node are not compatible with the stencil installer we use to create component packages. We saw the issue with a version of Node 6 and documentation states that version 6 is required. As a general note we should attempt to be running version 8 or higher of node.

# Package manager

We are using Yarn for development within our mono repo. Yarn provides the ability to have workspaces within your project and allows us to have multiple entry points. We will also be leveraging Lerna as an extension to our monorepo. Lerna provides additional functionality like running multiple packages during the build and versioning. But for the most part all we really need to be aware of as developers is the yarn package manager.

NPM has identified packages as a future enhancement and is on their roadmap but there is no release date as of (Sept 2019).
