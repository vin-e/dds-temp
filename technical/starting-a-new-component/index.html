# Starting a New Component

NOTE: This documentation is a moving target. In the, hopefully near, future we will be starting projects based on templates defined in the discover catapult project.

Each component is independently versioned and maintained. When starting a new component a new folder is created within the packages directory of the repository.

# Creating a New Component Project

# 1. navigate to the package directory
cd packages
# 2. init new stencil project
npm init stencil
# 3. select component project type
# 4. name the component with the following the format
#    of `dds-{component-name}`
# 5. confirm
# 6. navigate to new component
cd dds-{component-name}
# 7. update stencil
npm update
# 8. run the new project
npm run start

# Changing My-Component

During the init process the default stencil component template creates a component called my-component under the following folder of dds-{component-name}/src/components/my-component. This folder and it's contents should be renamed to your component name.

# Linking Other Components

In order to use another (existing) dds component within your new component you need in link and import. The linking process is done via yarn workspaces.

As an example if you wish to import dds-input into the new component you need to complete the following steps:

# 1 in the command line from the component folder
yarn workspace dds-{component-name} add dds-input@0.0.1
# 2 within your component `.tsx` file
`import 'dds-input'`

Once you have imported the component(s) you can now use the component within the render function.
