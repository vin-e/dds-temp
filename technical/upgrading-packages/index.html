# Upgrade Package Dependencies

In order to not fall behind with any dependency we have outlined the process below to update packages on a regular basis.

# Process

  1. Checkout the development branch and create a new branch named chore/upgrade-dev-packages (or something similar).
  2. At the root of the project run yarn upgrade-interactive --latest. NOTE: This command may ask that you update your yarn.lock file. In this case we would run yarn and then re-run the interactive upgrade.
  3. Build the projects via the yarn build command.
  4. Run all tests to ensure they are running via the yarn test command.
  5. Run a few of the packages to spot check that they still work as expected.
  6. Push the updates and submit a pull request for the changes.

# Issues

We will need to identify the cause of the issue. If it is a single package that caused the issue we need to rollback the dependency and we should continue the upgrade for the remainder of the packages.

# Known Issues

  • Excluding --latest from step 2 will not update the package.json file with the updated version numbers
  • Running yarn test in step 4 before running step 3 may cause issues during the tests.
